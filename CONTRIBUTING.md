# Contributing to mysqlpulse

## Development

```bash
make build    # compile binary to bin/mysqlpulse
make test     # run tests with race detector
make lint     # run golangci-lint
```

## Commits

Use conventional commits: `feat:`, `fix:`, `docs:`, `test:`, `refactor:`, `chore:`.

One-line summary, imperative mood, max 72 chars. Body only for "why".

## Pull Requests

- One PR per feature or fix
- All tests must pass
- Add tests for new functionality
